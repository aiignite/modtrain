## 1. 系统概述

### 1.1 系统功能

本系统是一款基于Python GUI的目标检测模型训练软件，主要用于：

| 功能模块 | 用途 |
|---------|------|
| **数据管理** | 导入图片/视频、标注查看、数据集准备 |
| **模型训练** | 使用YOLOv8进行目标检测模型训练 |
| **模型评估** | 评估训练好的模型性能（mAP、精确率、召回率） |
| **模型推理** | 用训练好的模型对新图片进行检测测试 |
| **实时监控** | 摄像头实时流处理（待完善） |

### 1.2 支持的检测类别

当前系统配置了以下检测类别：

| 类别ID | 类别名 | 中文含义 |
|--------|--------|----------|
| 0 | lp2d | 定位点2D |
| 1 | qr2d | 二维码 |

## 4. 模块一：数据管理

### 4.1 功能说明

数据管理模块用于：
1. 导入原始图片/视频数据
2. 查看已标注的目标框
3. 准备训练数据集（转换为YOLO格式）

### 4.2 操作流程

**完整工作流程**：
```
1. 导入图片 → data/imported/  (临时目录，用于标注)
2. 点击打开LabelImg → 自动打开imported目录进行标注
3. 标注完成保存 → XML存放在data/annotations/
4. 点击开始处理 → 自动分类：
   ├─ 有标注图片 → data/raw/ (原图) + data/processed/ (预览)
   ├─ 对应XML → data/annotations/
   ├─ 增强版本 → 自动生成对应增强图+XML
   └─ 无标注图片 → data/unlabeled/ (继续标注)
```

**目录说明**：
| 目录 | 用途 | 打开按钮 |
|------|------|----------|
| `data/imported/` | 待标注图片存放处 | 待标注 |
| `data/raw/` | 处理后的原图 | 原图 |
| `data/processed/` | 带标注框的预览图 | 预览 |
| `data/annotations/` | XML标注文件 | - |
| `data/unlabeled/` | 未标注的图片 | 未标注 |

#### 4.2.1 导入数据

```
步骤：
1. 点击"数据管理"标签页
2. 点击"导入图片"或"导入视频"
3. 选择要导入的文件
4. 导入的文件会存放在 data/raw/ 目录
```

**数据增强选项**：
可单独选择增强方式，**系统会自动转换标注框**

```
☑ 水平翻转 (推荐)   → 自动水平镜像标注框
☑ 垂直翻转 (推荐)   → 自动垂直镜像标注框
☑ 旋转 10°          → 自动计算旋转后外接矩形框
☑ 亮度变化 30        → 标注位置不变
```

**工作流程**：
```
1. 导入原始图片到 data/raw/
2. 用 LabelImg 标注原图 → 生成 data/annotations/*.xml
3. 勾选增强选项 → 点击开始处理
4. 系统自动生成：
   - 增强图片 → data/raw/xxx_hflip.jpg
   - 对应标注 → data/annotations/xxx_hflip.xml
```

**重要**：先标注原图，再做增强！标注会自动镜像/旋转
- 扩充训练数据量（约7倍）
- 提高模型泛化能力
- 减少过拟合风险
- 适用于样本量较少的情况

#### 4.2.2 查看标注

```
步骤：
1. 导入数据后，点击"开始处理"
2. 系统会读取 data/annotations/ 目录下的XML标注文件
3. 在预览区显示带标注框的图片
4. 使用"上一张"、"下一张"按钮切换查看
5. 信息区显示当前文件的标注详情
```

**标注信息格式**：
```
当前文件: LP-150-20-100mm.bmp
标注数量: 2 个

标注详情:
[1] lp2d: (100,50)-(200,150)
[2] qr2d: (300,100)-(400,200)
```

#### 4.2.3 准备训练数据

```
步骤：
1. 确保 data/raw/ 中有图片，data/annotations/ 中有对应XML标注
2. 点击"准备训练数据"
3. 系统执行以下操作：
   - 读取XML标注文件
   - 转换为YOLO格式（.txt标签文件）
   - 按比例拆分数据集（默认 8:1:1）
   - 生成 data.yaml 配置文件

输出目录：
- data/dataset/images/train/  (训练集 80%)
- data/dataset/images/val/    (验证集 10%)
- data/dataset/images/test/   (测试集 10%)
- data/dataset/labels/        (对应标签)
```

### 4.3 标注文件格式（XML）

系统支持Pascal VOC格式的XML标注文件：

```xml
<annotation>
    <filename>LP-150-20-100mm.bmp</filename>
    <size>
        <width>1280</width>
        <height>960</height>
    </size>
    <object>
        <name>lp2d</name>           <!-- 类别名称 -->
        <bndbox>
            <xmin>100</xmin>
            <ymin>50</ymin>
            <xmax>200</xmax>
            <ymax>150</ymax>
        </bndbox>
    </object>
    <object>
        <name>qr2d</name>
        <bndbox>...</bndbox>
    </object>
</annotation>
```

**注意事项**：
- 类别名称不区分大小写（lp2d、LP2D、Lp2D都可识别）
- 标注文件名需与图片名对应
- 坐标值为像素坐标

## 5. 模块二：模型训练

### 5.1 功能说明

使用YOLOv8进行目标检测模型训练，支持：
- 本地模型文件加载
- 实时训练进度显示
- 训练结果分析

### 5.2 参数说明

**支持的YOLO版本和模型大小**：

| YOLO版本 | 说明 | 模型大小 |
|---------|------|----------|
| **YOLOv3** | 经典版本，稳定可靠 | n/s/m/l/x |
| **YOLOv5** | 广泛使用，社区活跃 | n/s/m/l/x |
| **YOLOv8** | 最新版本，推荐使用 | n/s/m/l/x |

**模型大小说明**：

| 大小 | 名称 | 参数量 | 速度 | 精度 | 推荐场景 |
|------|------|--------|------|------|----------|
| **n** | nano | ~3M | 最快 | 较低 | 嵌入式/实时检测 |
| **s** | small | ~7M | 快 | 一般 | 平衡速度与精度 |
| **m** | medium | ~25M | 中 | 较好 | 多数场景推荐 |
| **l** | large | ~58M | 慢 | 高 | 需要高精度 |
| **x** | xlarge | ~68M | 最慢 | 最高 | 服务器离线处理 |

**训练参数**：

| 参数 | 含义 | 推荐值 | 说明 |
|------|------|--------|------|
| **训练轮次** | 训练迭代次数 | 50-300 | 太少欠拟合，太多过拟合 |
| **批次大小** | 每次训练样本数 | 4-16 | 根据显存调整 |
| **学习率** | 权重更新步长 | 0.001-0.01 | 过大会震荡，过小收敛慢 |
| **图像大小** | 输入图片分辨率 | 640 | 可选320/640/1280 |

### 5.3 操作流程

```
步骤：
1. 准备数据集（参考"数据管理"模块）
2. 选择数据集目录（data/dataset）
3. 选择模型：
   - 勾选"使用默认模型"：使用YOLO官方预训练模型
   - 取消勾选：点击"选择模型文件"选择本地.pt文件
4. 设置训练参数（参考上表推荐值）
5. 点击"开始训练"
6. 观察训练进度：
   - 进度条显示当前轮次
   - 实时显示损失值和mAP
7. 训练完成后点击"查看详细结果"
```

### 5.4 训练进度解读

**进度条**：
```
训练进度: 45% (45/100 轮)
损失值: 1.2345 | mAP@0.5: 0.6789
```

**关键指标**：
- **损失值 (Loss)**: 越低越好，训练过程应逐渐下降
- **mAP@0.5**: 平均精度，0.9以上表示优秀

**训练日志示例**：
```
使用本地模型: c:\...\yolo\yolov8n.pt
开始训练: epochs=100, batch_size=4
训练完成! 训练结果保存在: runs/train/exp
```

### 5.5 训练结果

训练完成后生成：

```
runs/train/exp/
├── weights/
│   ├── best.pt          # 最佳模型（验证集表现最好）
│   └── last.pt          # 最后一轮模型
├── results.png          # 训练曲线图表
├── results.csv          # 每轮详细数据
└── confusion_matrix.png # 混淆矩阵
```

**结果分析**：
```
mAP@0.5: 0.95        # 主要评估指标
mAP@0.5:0.95: 0.85   # 综合指标
精确率: 0.98         # 预测为正的样本中实际为正的比例
召回率: 0.92         # 正样本中被正确识别的比例
```

## 6. 模块三：模型评估

### 6.1 功能说明

对训练好的模型进行定量评估，生成：
- 评估指标（mAP、精确率、召回率）
- 混淆矩阵可视化

### 6.2 操作流程

```
步骤：
1. 切换到"模型评估"标签页
2. 系统自动加载：
   - 最新训练的模型: runs/train/exp/best.pt
   - 默认数据集: data/dataset
3. 点击"开始评估"
4. 等待评估完成（可能需要几分钟）
5. 查看结果：
   - 性能指标表格
   - 混淆矩阵图片
   - 错误分析与建议
```

### 6.3 结果解读

**评估建议**：

| mAP@0.5 | 评价 | 建议 |
|---------|------|------|
| > 0.9 | 优秀 | 模型表现很好 |
| 0.7-0.9 | 良好 | 可以继续训练提升 |
| 0.5-0.7 | 一般 | 建议增加数据或调整参数 |
| < 0.5 | 较差 | 需要检查数据和标注质量 |

**混淆矩阵解读**：
- 对角线：正确预测的数量
- 非对角线：误分类的数量
- 越亮表示数量越多

## 7. 模块四：模型推理

### 7.1 功能说明

使用训练好的模型对新图片进行目标检测，用于：
- 测试模型在新数据上的表现
- 批量处理测试图片
- 可视化检测结果

### 7.2 操作流程

```
步骤：
1. 切换到"模型推理"标签页
2. 选择模型：
   - 下拉框自动列出可用模型
   - 或点击"浏览..."选择其他模型
3. [可选] 勾选"启用多模型检测" - 同时加载yolov8n.pt检测80类物体
4. 点击"加载模型"
5. 选择测试图片：
   - "选择测试图片" - 多选图片
   - "选择测试目录" - 批量处理
6. 点击"开始推理"
7. 查看结果：
   - 使用翻页按钮查看各图片
   - 信息区显示检测到的目标
8. 点击"查看结果目录"打开输出文件夹
```

**多模型检测功能**：
```
☐ 启用多模型检测 (同时加载COCO模型检测80类物体)

勾选后：
- 自定义模型检测: lp2d, qr2d (绿色系标注框)
- COCO模型检测: person, car, truck... (紫色系标注框，前缀"COCO:")

输出显示: [1/10] test.jpg - 自定义:2 COCO:3
```

### 7.3 输出结果

**输出目录**: `data/inference_results/`

**推理参数**:
```
☐ 大图切片检测 (适合检测小目标)
  切片大小: [640 ▼]  可选: 640/800/1024/1280
  NMS阈值: [0.3] 越小去重越严格，减少误判

置信度阈值: 默认 0.25（可调节）
  - 降低阈值 = 检测更多目标（可能有误报）
  - 提高阈值 = 减少误报（可能漏检）
```

**切片检测减少误判的技巧**：
```
问题: 切片检测误判多
原因: 
  - 目标被切片边界切割，局部特征被误判
  - 缺少全局上下文信息

解决方案:
1. 增大切片大小
   640 → 800 或 1024（保留更多上下文）

2. 提高置信度阈值
   0.25 → 0.4 或 0.5（过滤低置信度误判）

3. 降低NMS阈值
   0.5 → 0.3 或 0.2（更严格去重）

推荐组合: 切片=1024, 置信度=0.4, NMS=0.3
```

**输出格式**:
```
原文件名: test.jpg
结果文件: test_result.jpg (带检测框)

推理时间: 245.8 ms        # 单图处理总时间
推理次数: 6 次            # 切片检测=切分数，单图=1

检测信息:
[1] lp2d (置信度: 0.95)
[2] qr2d (置信度: 0.88)
```

**日志输出**:
```
[1/10] test1.jpg - 检测到 8 个目标 [6次, 245.8ms]
[2/10] test2.jpg - 自定义:3 COCO:2 [1次, 45.2ms]
```

**用途**：
- 分析切片检测效率（次数 vs 时间）
- 对比不同图片的处理耗时
- 评估切片大小对性能的影响

**输出图片特点**:
- 绿色框: lp2d (定位点)
- 蓝色框: qr2d (二维码)
- 标签显示: 类别名 + 置信度

## 8. 模块五：实时监控

### 8.1 功能说明

通过摄像头实时进行目标检测，支持：
- 单模型检测（自定义训练的模型）
- 多模型检测（自定义模型 + COCO预训练模型）
- 实时FPS、检测时间、类别统计显示
- 异常检测与报警功能

### 8.2 操作流程

```
步骤：
1. 切换到"实时监控"标签页
2. 选择摄像头（系统会自动检测可用摄像头）
3. 选择模型：
   - 下拉框列出训练好的模型
   - 或点击"浏览..."选择其他模型
4. [可选] 勾选"启用多模型检测"
5. 点击"开始监控"
6. 观察实时检测结果
7. 点击"停止监控"结束
```

### 8.3 界面说明

**视频显示区**：
- 显示摄像头实时画面
- 检测到的目标用彩色框标注
- 标签显示类别名和置信度

**统计信息区**：
```
FPS: 25                    # 每秒处理帧数
检测时间: 35.42 ms         # 单帧处理时间
自定义: 3  COCO: 2         # 各模型检测数量
不良率: 100%               # 检测到目标的帧占比
检测类别: 龙贝码:3, QR码:2  # 各类别统计（中文显示）
```

### 8.4 多模型检测功能

```
☐ 启用多模型检测
  - 自定义模型: 检测训练的目标类别 (绿色框)
  - COCO模型: 检测80类常见物体 (紫色框，前缀COCO:)

显示格式:
  自定义: 3  COCO: 2        # 分别统计
  检测类别: 龙贝码:3, 人:1, 汽车:1
```

### 8.5 类别映射配置

所有检测到的类别名通过 `config/class_mapping.json` 自动翻译为中文：

| 英文类别 | 中文显示 | 来源 |
|---------|---------|------|
| lp2d | 龙贝码 | 自定义模型 |
| qr2d | QR码 | 自定义模型 |
| person | 人 | COCO模型 |
| car | 汽车 | COCO模型 |
| ... | ... | ... |

### 8.6 报警功能说明

当检测到目标时，系统内部会标记为"异常"，此功能可用于：
- 触发声音/灯光报警
- 记录异常帧
- 统计异常频率

当前版本为静默模式，如需报警可扩展。

## 9. 常见问题与解决方案

### 9.1 数据管理问题

**Q: 导入图片后看不到标注？**
```
A: 检查以下几点：
   1. data/annotations/ 目录下是否有对应XML文件
   2. XML文件名是否与图片名一致
   3. XML中的<name>标签是否是lp2d或qr2d
```

**Q: "准备训练数据"按钮没反应？**
```
A: 确保：
   1. data/raw/ 目录下有图片文件
   2. data/annotations/ 目录下有对应XML标注
   3. 检查日志输出是否有错误提示
```

### 9.2 模型训练问题

**Q: 训练时提示CUDA out of memory？**
```
A: 解决方案：
   1. 减小批次大小（batch_size）
   2. 减小图像大小（img_size: 640→320）
   3. 使用CPU训练（较慢）
```

**Q: 训练损失不下降，mAP一直为0？**
```
A: 可能原因：
   1. 标注文件格式错误
   2. 类别名称不匹配
   3. 数据集太小
   
   解决：
   1. 检查data.yaml中的names是否与标注一致
   2. 增加训练样本（建议至少50张/类）
   3. 检查标注坐标是否在图片范围内
```

**Q: 如何继续训练已有的模型？**
```
A: 在模型选择时：
   1. 取消勾选"使用默认模型"
   2. 点击"选择模型文件"
   3. 选择 runs/train/exp/weights/last.pt
   4. 设置较少的训练轮次继续训练
```

### 9.3 模型推理问题

**Q: 检测不到目标？**
```
A: 检查：
   1. 模型是否加载成功
   2. 测试图片是否与训练数据类似
   3. 目标是否太小或太模糊
   4. 检查日志是否有错误
```

### 9.4 路径问题

**Q: 程序找不到文件？**
```
A: 确保：
   1. 从项目根目录运行程序
   2. 命令: python main.py
   3. 不要双击.py文件运行
```

## 10. 最佳实践建议

### 10.1 数据集准备

| 建议 | 说明 |
|------|------|
| 样本数量 | 每类至少50张，推荐100+ |
| 样本多样性 | 不同角度、光照、背景 |
| 标注质量 | 仔细检查每个标注框 |
| 类别均衡 | 各类别样本数尽量接近 |

### 10.2 训练参数推荐

**初次训练**：
```
训练轮次: 100
批次大小: 4 (根据显存调整)
学习率: 0.01
图像大小: 640
```

**微调训练**：
```
训练轮次: 50
学习率: 0.001
使用: best.pt 继续训练
```

### 10.3 模型选择

| 模型 | 速度 | 精度 | 适用场景 |
|------|------|------|----------|
| yolov8n.pt | 快 | 低 | 快速测试、CPU推理 |
| yolov8s.pt | 中 | 中 | 平衡选择 |
| yolov8m.pt | 慢 | 高 | 高精度要求 |

### 10.4 工作流建议

```
推荐流程:
1. 准备50张标注图片
2. 训练100轮
3. 评估模型性能
4. 用模型推理测试集
5. 分析误分类样本
6. 补充标注困难样本
7. 重复步骤2-6直到满意
```
