# 生产过程人工操作监控系统 - 详细规格说明书

## 一、系统概述

### 1.1 项目背景

本系统是一个基于姿态识别和物体检测的生产过程人工操作监控解决方案，旨在：
- 学习并记录标准操作流程
- 实时监控生产过程中的人工操作
- 检测操作步骤错误、顺序错误、遗漏步骤等问题
- 提供实时告警和数据追溯功能

### 1.2 目标用户

- 生产线操作员
- 质量管理人员
- 工艺工程师
- 现场管理人员

### 1.3 部署平台

- 嵌入式平台：RK3588 (ARM64)
- 操作系统：Ubuntu 20.04 / Debian 12
- 运行环境：Python 3.10 + ONNX Runtime

## 二、功能模块设计

### 2.1 核心功能模块

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| **学习模式** | 录制标准操作流程，提取动作特征 | 高 |
| **监控模式** | 实时检测操作步骤，验证正确性 | 高 |
| **步骤编辑** | 创建、编辑、删除操作流程配置 | 高 |
| **告警管理** | 多级别告警、告警记录 | 中 |
| **数据存储** | 本地存储、导出功能 | 中 |
| **报表分析** | 操作统计、错误分析 | 低 |

### 2.2 学习模式

#### 2.2.1 功能描述

学习模式用于录制标准操作流程，包括以下步骤：

1. **创建流程**
   - 输入流程名称、描述、版本
   - 选择操作场景（PCB装配、焊接、检测等）

2. **录制操作**
   - 实时显示摄像头画面
   - 录制操作过程视频
   - 自动检测并记录关键动作
   - 手动标记步骤边界

3. **提取特征**
   - 提取关节角度序列
   - 计算动作特征
   - 生成标准模板

4. **保存流程**
   - 保存流程配置（JSON）
   - 保存动作模板
   - 保存示例视频

### 2.3 监控模式

#### 2.3.1 功能描述

监控模式用于实时检测操作正确性：

1. **加载流程**
   - 选择要监控的流程
   - 加载流程配置和模板

2. **实时监控**
   - 摄像头视频流输入
   - 实时姿态估计
   - 实时物体检测
   - 动作识别
   - 步骤验证

3. **告警处理**
   - 检测到错误时触发告警
   - 记录告警信息
   - 显示告警提示

4. **数据记录**
   - 记录操作过程
   - 保存异常帧
   - 生成操作日志

### 2.6 告警管理

#### 2.6.1 告警级别

| 级别 | 名称 | 触发条件 | 告警方式 |
|------|------|----------|----------|
| **INFO** | 提示 | 步骤完成 | 界面提示 |
| **WARNING** | 警告 | 时间超限、动作异常 | 声音+界面 |
| **ERROR** | 错误 | 顺序错误、遗漏步骤 | 声音+界面+记录 |
| **CRITICAL** | 严重 | 安全违规 | 声音+界面+停机 |

#### 2.6.2 告警处理流程

```
检测到异常
    │
    ▼
判断告警级别
    │
    ├─ INFO ──────────▶ 界面显示
    │
    ├─ WARNING ───────▶ 界面显示 + 声音提示
    │
    ├─ ERROR ─────────▶ 界面显示 + 声音提示 + 记录日志
    │                      │
    │                      ▼
    │                   保存截图/视频
    │
    └─ CRITICAL ───────▶ 界面显示 + 声音告警 + 停机信号
                         │
                         ▼
                      多渠道通知(短信/邮件)
```

### 5.1 动作识别算法

# 基于规则的动作识别
def recognize_action(keypoints, objects):
    """
    输入: 关键点数组, 检测到的物体
    输出: 动作类型, 置信度
    
    规则逻辑:
    1. 计算手臂角度 (肘关节)
    2. 计算手臂伸展程度
    3. 判断手部位置 (前方/下方)
    4. 判断是否有物体在手中
    5. 应用规则树判断动作类型
    """
    
    # 提取特征
    left_arm_angle = calculate_angle(kp[5], kp[7], kp[9])  # 左肩-左肘-左腕
    right_arm_angle = calculate_angle(kp[6], kp[8], kp[10]) # 右肩-右肘-右腕
    
    arm_height = (shoulder_y - wrist_y) / image_height
    
    # 判断逻辑
    if arm_angle > 140 and arm_height > 0.6:
        if has_object_in_hand(objects):
            return "PICKING", 0.85
        else:
            return "REACHING", 0.75
    
    elif arm_angle < 60 and arm_height < 0.4:
        if has_object_in_hand(objects):
            return "PLACING", 0.80
        else:
            return "RETRACTING", 0.70
    
    elif hip_angle > 160 and arm_height < 0.4:
        return "INSPECTING", 0.75
    
    else:
        return "IDLE", 0.90

### 5.2 步骤验证算法

```python
# 有限状态机验证
class StepValidator:
    def __init__(self, process_config):
        self.current_step = None
        self.step_start_time = None
        self.state = "IDLE"
    
    def validate(self, action, timestamp):
        # 状态转换逻辑
        if self.state == "IDLE":
            if action in current_step.expected_actions:
                self.state = "IN_PROGRESS"
                self.step_start_time = timestamp
        
        elif self.state == "IN_PROGRESS":
            duration = timestamp - self.step_start_time
            
            # 检查超时
            if duration > current_step.max_duration:
                self.trigger_alarm("TIMEOUT")
                return "TIMEOUT"
            
            # 检查动作是否匹配
            if action in current_step.expected_actions:
                if duration >= current_step.min_duration:
                    # 步骤完成
                    if has_next_step():
                        self.current_step = next_step
                        self.state = "WAITING"
                    else:
                        self.state = "COMPLETED"
            else:
                # 动作不匹配
                if duration > current_step.min_duration:
                    self.trigger_alarm("ACTION_MISMATCH")
        
        return self.state
```
